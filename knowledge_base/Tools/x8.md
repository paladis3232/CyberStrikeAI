# x8

## Overview
- Tool name: `x8`
- Enabled in config: `true`
- Executable: `x8`
- Default args: none
- Summary: Hidden parameter discovery tool

## Detailed Description
X8 is a tool for discovering hidden parameters in web applications.

**Key Features:**
- Hidden parameter discovery
- Support for multiple HTTP methods
- Custom wordlists
- Fast scanning

**Use Cases:**
- Parameter discovery
- Web application security testing
- Bug bounty reconnaissance
- Security testing

## Parameters
### `url`
- Type: `string`
- Required: `true`
- Flag: `--url`
- Format: `flag`
- Description: Target URL

### `wordlist`
- Type: `string`
- Required: `false`
- Flag: `--wordlist`
- Format: `flag`
- Default: `/usr/share/wordlists/x8/params.txt`
- Description: Parameter wordlist

### `method`
- Type: `string`
- Required: `false`
- Flag: `--method`
- Format: `flag`
- Default: `GET`
- Description: HTTP method

### `body`
- Type: `string`
- Required: `false`
- Flag: `--body`
- Format: `flag`
- Description: Request body

### `headers`
- Type: `string`
- Required: `false`
- Flag: `--headers`
- Format: `flag`
- Description: Custom request headers

### `additional_args`
- Type: `string`
- Required: `false`
- Format: `positional`
- Description: Additional x8 parameters. Used to pass x8 options not defined in the parameter list.

## Invocation Template
```bash
x8 <url> <wordlist> <method> <body> <headers> <additional_args>
```

## Model Usage Guidance
- Use this tool only within authorized scope.
- Prefer the narrowest target/argument set before broad scans.
- For long outputs, store results and summarize key findings.
- Validate parameter formats before execution to reduce command errors.
